Exit nodes

Obligatory route

Every node that want to join your tailscale network must have Tailscale Client ‚Üó installed (see regular nodes).

Installation#

Supposing a GNU/Linux Debian 13 Trixie operating system, you can install it with curl -fsSL https://tailscale.com/install.sh | sh.

Configuration#

Then you need to join the network using a running headscale server, but this time you need to declare the node as an exit one.

tailscale up \
    --login-server=https://hs.yourdomain.app \
    --advertise-exit-node \
    --accept-routes \
    --accept-dns=true \
    --hostname=your-node-name
shell

The first time you need to accept the new node following the instructions shown.

Then you can check the list of registered nodes on headscale server with headscale nodes list.

The exit node should be listed also as route: headscale nodes list-routes.

Firewall setup#

If your node is exposed to Internet (ie. a VPS with a public IP) and you want to control open ports, you can allow udp traffic on port 41641. Using ufw: ufw allow 41641/udp.

Incoming traffic#

If you need to forward incoming traffic to another node on tailscale network, you need to allow forwarding at kernel and firewall level, and incoming traffic on tailscale network interface.

  1. Allow forwarding (kernel). 
    echo 'net.ipv4.ip_forward = 1' | tee -a /etc/sysctl.d/99-tailscale.conf
echo 'net.ipv6.conf.all.forwarding = 1' | tee -a /etc/sysctl.d/99-tailscale.conf
sysctl -p /etc/sysctl.d/99-tailscale.conf
    shell
  2. Allow forwarding (firewall): ufw default allow forward
  3. Allow tailscale incoming traffic: ufw allow in on tailscale0

Now you have two options: use a reverse proxy or configure a NAT.

  • Reverse proxy
  • NAT

Incoming web traffic (on ports 80 or 443) can be managed by a reverse proxy (ie. Caddy) and forwarded to a node using tailscale interface.

/etc/caddy/Caddyfile
public.yourdomain.app {
    reverse_proxy 100.64.0.X:8080
}
txt

Then apply changes: systemctl restart caddy.

For generic tcp/udp traffic you can forward an exposed port of the node to a port of another node on tailscale network.

/etc/ufw/before.rules
*nat
:PREROUTING ACCEPT [0:0]
-A PREROUTING -p tcp --dport 9999 -j DNAT --to-destination 100.64.0.X:9999
-A POSTROUTING -p tcp -d 100.64.0.X --dport 9999 -j MASQUERADE
COMMIT
txt

CAUTION

This *nat block should be on top of the file before *filter block! And don‚Äôt forget the COMMIT!

Of course the port must be open: ufw allow 9999/tcp.

Then apply changes: systemctl restart ufw.

Kernel tweaks#

Update the Generic Receive Offload (GRO) settings if needed.

  1. Install networkd-dispatcher: apt install networkd-dispatcher
  2. Create new rule. 
    printf \
    '#!/bin/sh\n\nethtool -K %s rx-udp-gro-forwarding on rx-gro-list off \n' \
    "$(ip -o route get 8.8.8.8 | cut -f 5 -d " ")" | \
    tee /etc/networkd-dispatcher/routable.d/50-tailscal
    shell
  3. Make it executable: chmod 755 /etc/networkd-dispatcher/routable.d/50-tailscale
  4. Run it: /etc/networkd-dispatcher/routable.d/50-tailscale

See the official documentation ‚Üó for details.
